# Content Review

## 审校目标

本文件记录 1.0 候选版的题目和结果文案审校结论。目标是确保内容自然、克制、适合中文用户，并符合产品边界：

- 不使用受保护的人格测试名称或表达。
- 不模仿人格类型测试。
- 不进行医学、心理、法律或职业诊断。
- 不承诺科学测量结论。
- 不把结果写成固定身份或终身标签。

## 题库审校

当前题库：

- 总题数：48。
- Mind / 认知：12 题。
- Body / 身体：12 题。
- Spirit / 意义：12 题。
- Vocation / 事业：12 题。
- 反向计分题：12 题，每个象限 3 题。

审校结论：

- 题干整体为中文日常表达，没有论文式或临床式措辞。
- 题目围绕自我理解、生活系统和行动习惯，不做疾病、障碍或人格诊断判断。
- 反向题分布均衡，可降低机械作答风险。
- Body 题目使用“精力、作息、恢复、身体活动”等发展性表达，避免把身体状态写成健康诊断。
- Spirit 题目聚焦价值、欲望、方向和外部评价驱动，避免宗教化或玄学化表达。
- Vocation 题目聚焦能力、作品、反馈和价值转化，避免承诺收入或职业成功。

已调整：

- 将首页状态从“静态 MVP”更新为“1.0 候选”。
- 将结果模板 `Human 2.2` 的 Metatype 从固定身份感较强的表达改为 `The System in Motion`。
- 从 PRD、README、HANDOFF 中移除受保护人格测试名称和模仿式定位。

## 结果文案审校

审校范围：

- `data/result-templates.json`
- `data/recommendations.json`
- `data/site-content.json`
- 结果页展示组件中的固定标题
- 分享卡片底部说明

审校结论：

- 结果文案以“当前状态快照”为主，不把用户固定成某类人。
- Core Problem 聚焦当前系统卡点，不使用绝对化或恐吓性表达。
- Cross-Quadrant Dynamics 解释象限互动，避免神秘化。
- Immediate Next Action 和 7/30/90 天建议具体可执行，不承诺治疗、治愈、财富或职业结果。
- 分享卡片保留记忆点，但底部明确“不是人格标签”。

## 仍需真实用户确认

1. 用 10 到 20 个样例答案检查阶段阈值是否符合产品直觉。
2. 找 3 到 5 位真实用户完成测评，记录哪些题目难懂、冒犯或过于抽象。
3. 用手机浏览器检查分享卡片下载和社交平台展示效果。
4. 检查英文 Metatype 对中文用户是否有帮助；如果理解成本高，可在 1.1 改为中英文并列。

## 审校结论

当前内容达到 1.0 候选版发布前的内部内容审校标准。1.0 正式发布前，仍建议完成小样本真实用户阅读反馈和阶段阈值校准。
