# Decisions

- 日期：2026-06-22
- 决策：使用根级 `app/` 目录，不使用 `src/`。
- 原因：用户指定的项目结构希望 `app/` 位于项目根目录。
- 替代方案：使用 `src/app`。
- 影响：文件路径更接近需求，Next.js 仍然完全支持。

- 日期：2026-06-22
- 决策：第一版结果保存在 localStorage，并保留 `/api/submit`。
- 原因：第一版不接数据库，但未来需要平滑迁移到 Supabase。
- 替代方案：只做纯前端，不创建 API。
- 影响：未来接入持久化时可复用服务端结果构建逻辑。

- 日期：2026-06-22
- 决策：计分规则采用可解释启发式阈值。
- 原因：需求给出方向性规则，但没有精确阈值；MVP 需要稳定可测的默认实现。
- 替代方案：让所有阈值配置化到 JSON。
- 影响：当前逻辑集中在 `lib/scoring.ts`，后续可迁移为配置驱动。

- 日期：2026-06-22
- 决策：页面不展示原始分数。
- 原因：需求要求结果页展示状态而不是分数，减少用户过度量化。
- 替代方案：显示分数和雷达图。
- 影响：UI 更克制，调试时可通过测试和内部对象查看分数。

- 日期：2026-06-22
- 决策：视觉采用浅色、克制、低饱和配色。
- 原因：产品定位是自我发展评估，不是娱乐测试或夸张营销页。
- 替代方案：暗色仪表盘或强渐变视觉。
- 影响：中文阅读体验更平稳，移动端负担更低。

- 日期：2026-06-22
- 决策：移除 `next/font/google`，改用系统中文字体栈。
- 原因：本地 Turbopack 开发服务器出现字体内部模块解析错误；系统字体对中文体验也更稳定。
- 替代方案：切换 dev 脚本到 webpack，或继续排查 Turbopack 字体依赖。
- 影响：减少开发环境不稳定性，不再依赖 Google 字体加载。

- 日期：2026-06-22
- 决策：新增 `pnpm validate:data` 数据校验命令。
- 原因：题库和模型数据未来会频繁调整，需要一个低成本检查入口保证题量、字段和覆盖范围不漂移。
- 替代方案：只依赖人工检查或测试用例间接发现。
- 影响：产品人员改 JSON 后可以先运行数据校验，再交给开发者接入。

- 日期：2026-06-22
- 决策：首页产品文案移入 `data/site-content.json`。
- 原因：减少组件中的大段文案硬编码，让产品文案和展示组件解耦。
- 替代方案：继续写在 `components/AssessmentIntro.tsx`。
- 影响：首页文案可由产品人员直接改 JSON，组件更偏展示职责。

- 日期：2026-06-22
- 决策：答题页移动端增加底部安全留白。
- 原因：本地开发环境的 Next.js 开发工具浮层可能遮挡左下角“上一题”按钮。
- 替代方案：忽略开发环境浮层，或要求手动关闭浮层。
- 影响：移动端和本地验收时底部按钮更不容易被遮挡。
